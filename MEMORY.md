# MEMORY.md

## Communication

* Prefer clear, direct explanations.
* Prefer practical steps and commands.
* Avoid unnecessary verbosity.

## Coding

* Beginner-friendly but professional Python.
* Readability over clever abstractions.
* Use pathlib, argparse, descriptive names.
* Prefer small testable functions.

## Comments

* Explain what and why.
* Heavy but natural technical commenting.
* Avoid generic AI-sounding comments.

## Documentation

* Professional research tone.
* Explain purpose, assumptions, inputs, outputs, and evaluation.

## Repository

* Stable code in scripts/core/.
* New work in scripts/experimental/.
* Preserve working code.

## Research

* Understand results before replacing models.
* Prioritize analysis, debugging, and validation before training new architectures.

## Updating Memory

Only store durable preferences.
Do not store temporary tasks, experiment results, or one-time instructions.
